# 配置指南


注：表情包请自行配置，在assets文件夹新建stickers，里面放各种情绪的文件夹（沉默/害羞/开心/委屈/无奈/头疼）

## 必填项清单

部署前必须修改以下字段，其他保持默认即可。

---

## 1. NapCat 连接（`qq` 节）

```yaml
qq:
  host: 127.0.0.1
  port: 3001
```

打开 NapCat 管理界面，确认 WebSocket 服务端的端口号，填入 `port`。默认 3001 一般不用改。

---

## 2. 主 LLM（`llm` 节）

```yaml
llm:
  api_key: YOUR_LLM_API_KEY
```

**推荐：DeepSeek**
1. 访问 [platform.deepseek.com](https://platform.deepseek.com) 注册
2. 创建 API Key，复制后填入 `api_key`
3. `model` 填 `deepseek-chat`，`base_url` 填 `https://api.deepseek.com`

其他兼容 OpenAI 格式的服务（Claude、Qwen、Moonshot 等）同理，改对应的 `api_key` / `model` / `base_url` 即可。

---

## 3. 视觉模型（`vision` 节）—— 识别你发的图片

```yaml
vision:
  api_key: YOUR_VISION_API_KEY
```

三选一，按需选一个免费的：

| Provider | 注册地址 | 推荐模型 |
|---|---|---|
| **GLM（推荐，国内直连）** | [bigmodel.cn](https://bigmodel.cn) | `glm-4v-flash` |
| **Gemini** | [aistudio.google.com](https://aistudio.google.com)（需代理）| `gemini-2.0-flash` |
| **OpenAI** | [platform.openai.com](https://platform.openai.com)（付费）| `gpt-4o-mini` |

注册后创建 API Key，填入 `vision.api_key`，同时把 `vision.provider` 改成对应名称。

> 不需要图片识别就把 `vision.enabled` 改成 `false`。

---

## 4. 管理面板密钥（`admin` 节）

```yaml
admin:
  secret_key: YOUR_ADMIN_SECRET
```

自己随便设一个复杂字符串（比如 `MyBot@2024!`），用于登录 `http://127.0.0.1:8080` 管理面板。

---

## 5. 主人 QQ 号与生日（`notify` 和 `scheduler` 节）

```yaml
notify:
  target_qq: '你的QQ号'

scheduler:
  owner_id: '你的QQ号'
  owner_birthday: "04-24"   # 格式 MM-DD
```

填你自己的 QQ 号（带引号）。机器人上线通知和所有主动消息都发给这个 QQ。

`owner_birthday` 用于生日触发（前夜预热、零点告白、下午关心、夜间收尾），格式固定为 `MM-DD`，例如 4月24日填 `"04-24"`。

---

## 6. 角色卡（`character` 节）

```yaml
character:
  default: 角色名.txt    # characters/ 目录下的角色卡文件名
  name: 角色名           # 调度器触发消息里使用的名字，和角色卡文件名保持一致
```

在 `characters/` 目录下放一个 `.txt` 文件，文件名填入 `character.default`，`character.name` 填角色的称呼（用于拼接主动消息的旁白）。

**角色卡格式**（纯文本，无固定结构，LLM 直接读取作为 system prompt 的一部分）：

```我的character/txt示例文本里有写，自己写或者用ai跑

【基本设定】
【背景故事】
...（自由发挥）
```

没有强制格式，写得越具体，角色表现越稳定。建议把说话风格、禁止行为、背景故事都写进去。

---

## 7. Obsidian 日记路径（`diary` 节）—— 可选

```yaml
diary:
  obsidian_path: "C:\\path\\to\\your\\obsidian\\vault"
```

如果你在 Obsidian 里写日记，填入 vault 中存放日记的文件夹绝对路径（不是 vault 根目录，是具体的日记子目录）。机器人会定期读取最近几天的 `.md` 文件作为上下文。

Windows 路径用双反斜杠 `\\` 或正斜杠 `/` 均可：
- `"E:\\obsidian_note\\Psychology"`
- `"E:/obsidian_note/Psychology"`

不使用此功能就留空或删掉这一节，`diary_reminder` 和 `diary_inject` 调度器会自动跳过。

---

## 8. 语音合成（`tts` 节）—— 可选

不需要语音就把 `tts.enabled` 改成 `false`，跳过此节。

需要语音则：
1. 部署 GPT-SoVITS（整合包版本），启动 Gradio API 服务
2. 把 `tts.api_url` 改成对应地址（默认 `http://127.0.0.1:9872`）
3. 准备参考音频（WAV 格式），填入 `tts.ref_audio` 的绝对路径
4. 如果有多个情绪音频，分别填入 `tts.emotions` 各项的 `ref_audio`；没有就留空 `''`

`tts.probability` 控制触发概率（0.0～1.0），默认 `0.1` 即每条消息 10% 概率发语音。

---

## 9. 代理（`proxy` 节）—— 可选

如果你在国内访问 Gemini / OpenAI 需要代理：

```yaml
proxy:
  enabled: true
  http: http://127.0.0.1:你的代理端口
  https: http://127.0.0.1:你的代理端口
```

使用国内服务（DeepSeek、GLM）则保持 `enabled: false`。

---

## 10. Apple Watch 健康数据接入 —— 可选

心率异常提醒、睡眠感知等功能需要 Apple Watch 推送数据。

详见 [Watch配置指南.md](Watch配置指南.md)。

---

## 快速上手步骤

1. 复制 `config.yaml`，按上面说明填写必填项
2. 在 `characters/` 目录放入角色卡 `.txt` 文件，修改 `character.default` 和 `character.name`
3. 启动 NapCat，确保 QQ 已登录
4. 运行 `python main.py`
5. 浏览器打开 `http://127.0.0.1:8080`，用 `admin.secret_key` 登录管理面板
